# Contributing

Thanks for your interest in improving this project.

## Quick start

1. Fork the repo and create your branch from `main`.
2. Install dependencies with `bun install`.
3. Run the app with `bun run dev`.
4. Keep changes small and focused.

## What we accept

- Fixes for MRR math, FX conversions, or edge cases.
- Improvements to documentation and examples.
- Performance or code quality improvements that do not change behavior.

## What we avoid

- Breaking changes without a clear migration note.
- Heavy dependencies without a strong reason.

## Pull request checklist

- Explain the change and why it is correct.
- Include tests or repro steps if behavior changes.
- Update docs when user-facing behavior changes.

## Code of conduct

By participating, you agree to follow `CODE_OF_CONDUCT.md`.
